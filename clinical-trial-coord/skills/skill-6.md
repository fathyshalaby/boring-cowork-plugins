# Data Management & Source Documentation
Maintain impeccable data quality through proper source documentation and CRF completion:

### Source Document Requirements
- The source document is the ORIGINAL record of clinical data: medical chart, lab report, ECG tracing, imaging report, patient diary.
- Every data point entered into the CRF (Case Report Form) or EDC (Electronic Data Capture) system must be traceable to a source document.
- Source documents must be: legible, contemporaneous (recorded at the time of the event), original, accurate, and complete.

### EDC Best Practices
- Enter data within 24-48 hours of the visit (or per protocol-specified timeline).
- Respond to data queries within the sponsor's required timeframe (typically 5-10 business days).
- Never fabricate data. If a value is missing, document why and note it as missing.
- Use the edit check and query management features of the EDC system—they catch errors before monitoring visits.

### ALCOA+ Principles
| Principle | Meaning |
|-----------|---------|
| Attributable | Who performed/recorded the action and when |
| Legible | Readable, permanent (no pencil) |
| Contemporaneous | Recorded at the time of the event |
| Original | First recording of data (or certified copy) |
| Accurate | Correct and truthful |
| +Complete | All activities documented |
| +Consistent | Dates and times are logical |
| +Enduring | Stored on durable media, not easily altered |
| +Available | Accessible for review, audit, and inspection |
