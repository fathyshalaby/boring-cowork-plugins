# Security Frameworks & Standards
Apply industry security frameworks to organizational security programs:

### Major Frameworks
| Framework | Scope | Best For |
|-----------|-------|---------|
| NIST CSF 2.0 | Identify, Protect, Detect, Respond, Recover, Govern | Any organization — voluntary, flexible |
| ISO 27001 | Information Security Management System (ISMS) | Organizations seeking certification |
| CIS Controls v8 | 18 prioritized security controls | Practical implementation prioritization |
| SOC 2 | Trust Service Criteria (Security, Availability, etc.) | SaaS/service companies |
| PCI DSS v4.0 | Payment card data protection | Any org processing card data |
| HIPAA Security Rule | Electronic health information | Healthcare organizations |
| FedRAMP | Cloud services for federal government | Cloud service providers |

### NIST CSF 2.0 Functions
1. **Govern**: Establish and monitor risk management strategy, expectations, and policy.
2. **Identify**: Asset management, risk assessment, supply chain risk.
3. **Protect**: Access control, awareness training, data security, platform security.
4. **Detect**: Continuous monitoring, adverse event analysis.
5. **Respond**: Incident management, analysis, mitigation, reporting.
6. **Recover**: Recovery planning, improvements, communications.

### Framework Selection
- Start with CIS Controls for quick wins and practical security improvements.
- Use NIST CSF for comprehensive program structure.
- Pursue ISO 27001 or SOC 2 when customers or contracts require certification.
