# Diagram & Visual Documentation
Use diagrams and visuals to clarify complex concepts:

### When to Use Visuals
- Architecture overviews and system diagrams.
- Workflows and processes with multiple paths.
- Data models and relationships.
- Network topology.
- State machines and lifecycle diagrams.
- UI walkthroughs (annotated screenshots).

### Diagramming Tools
| Tool | Format | Best For |
|------|--------|---------|
| Mermaid | Text-to-diagram (in Markdown) | Docs-as-code, simple diagrams |
| PlantUML | Text-to-diagram | UML diagrams, sequence diagrams |
| draw.io (diagrams.net) | GUI/XML | Complex architecture diagrams |
| Excalidraw | GUI/JSON | Whiteboard-style, hand-drawn feel |
| Lucidchart | SaaS | Team collaboration, templates |
| D2 | Text-to-diagram | Modern syntax, layout engine |

### Diagram Best Practices
- Every diagram needs a title and a brief explanation in surrounding text.
- Use consistent styling (colors, shapes, fonts) across all diagrams.
- Keep diagrams simple — if it needs >15 elements, break into sub-diagrams.
- Text-based diagram source (Mermaid, PlantUML) is versionable in Git.
- Include alt text for accessibility.
