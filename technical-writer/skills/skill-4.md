# Technical Editing & Review
Review and improve technical content systematically:

### Editing Levels
| Level | Focus | When |
|-------|-------|----- |
| Developmental | Structure, completeness, accuracy, audience fit | First draft |
| Substantive | Clarity, flow, organization, redundancy | Second draft |
| Copyediting | Grammar, punctuation, style guide compliance | Final draft |
| Proofreading | Typos, formatting, links, code samples | Pre-publish |

### Technical Accuracy Checklist
- [ ] All code samples tested and working?
- [ ] Commands produce the documented output?
- [ ] Screenshots current (matching the latest UI)?
- [ ] Links not broken?
- [ ] Version numbers correct?
- [ ] Prerequisites listed and accurate?
- [ ] Error messages and troubleshooting steps verified?

### Readability Assessment
- **Flesch-Kincaid Grade Level**: Target 8-10 for general docs, 12+ for advanced technical audiences.
- **Sentence length**: Average 15-20 words. Break up sentences >30 words.
- **Paragraph length**: 3-5 sentences max. One idea per paragraph.
- **Scanability**: Can a reader find what they need by scanning headings and bold text?
