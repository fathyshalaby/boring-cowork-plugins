# Lump Sum Funding
Navigate the transition to Lump Sum grants:

### What is Lump Sum Funding?
A simplified financial model replacing actual cost reimbursement. The EC pays a fixed amount for the completion of Work Packages (WPs), removing the need to report actual costs, timesheets, or undergo financial audits.

### Pre-Award / Budgeting Phase
- You still build a detailed budget bottom-up using a specific Excel workbook provided by the EC. Evaluators use this spreadsheet to assess if the lump sum requested is reasonable.
- Costs must reflect local market rates and typical personnel costs.
- The budget is locked upon Grant Agreement signature. Evaluators can recommend reducing it if deemed inflated.

### Project Implementation Phase
- **Focus shifts completely from financial reporting to technical performance.** 
- Payment depends on the *completion* of WPs. If a WP is partially complete at the end of a reporting period, it is not paid until the next period when it is fully finished.
- Therefore, WP design in the proposal is critical: 
  - Avoid WPs that run the entire length of the project (e.g., split WP Management into "WP Management M1-M18" and "WP Management M19-M36").
  - Define clear, verifiable outcomes for each WP to prove completion to the Project Officer.
- Budget transfers between WPs are possible but require an amendment if the WP has already been paid or if it changes the technical work.
