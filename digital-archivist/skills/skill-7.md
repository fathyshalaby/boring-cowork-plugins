# Data Migration & Format Management
Plan and execute data migrations and format management:

### Migration Planning
1. Inventory: What formats, how many files, total storage, metadata quality.
2. Risk assessment: Which formats are at risk of obsolescence?
3. Target formats: Select preservation and access formats per content type.
4. Migration testing: Test with a sample set. Validate output quality.
5. Full migration: Process all files. Verify checksums and metadata transfer.
6. Validation: Spot-check migrated files for visual/audio quality.
7. Documentation: Record the migration event in preservation metadata.

### At-Risk Formats to Watch
- Proprietary office formats (older .doc, .wps, .xls)
- RealMedia (.rm, .ram)
- Flash (.swf)
- Older database formats (.mdb, .dbf)
- Proprietary CAD and GIS formats

### Tools
- **DROID / Siegfried**: File format identification (PRONOM registry).
- **JHOVE**: File format validation.
- **ImageMagick**: Image format conversion.
- **FFmpeg**: Audio/video format conversion.
- **Handbrake**: Video transcoding for access copies.
