# Form-Based Codes & Design Standards
Implement form-based codes and design standards:

### Form-Based Code vs. Conventional Zoning
| Aspect | Conventional Zoning | Form-Based Code |
|--------|--------------------|-----------------|
| Primary control | Use (what happens inside) | Form (building shape, placement) |
| Mixed use | Typically separated | Encouraged |
| Predictability | Variable outcomes | Predictable physical form |
| Flexibility | Rigid use categories | Flexible uses within form |
| Public realm | Often neglected | Primary focus |

### Form-Based Code Components
- **Regulating plan**: Maps the transect zones or character areas.
- **Building form standards**: Setbacks, height, massing, frontage types.
- **Frontage types**: Shopfront, stoop, forecourt, gallery, arcade, terrace.
- **Public space standards**: Street cross-sections, sidewalks, street trees, plaza design.
- **Architectural standards**: Materials, windows, entrances, parking screening.
- **Signage standards**: Type, size, placement, illumination per zone/frontage.

### SmartCode / Transect
- T1 (Natural Zone) → T2 (Rural) → T3 (Sub-Urban) → T4 (General Urban) → T5 (Urban Center) → T6 (Urban Core).
- Each zone has appropriate building forms, street types, and uses.
- Ensures a coherent transition from rural to urban.
