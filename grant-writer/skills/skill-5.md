# Needs Assessment & Logic Models
Build compelling needs statements and program logic models:

### Needs Assessment
- Use data to quantify the problem: census data, health statistics, education metrics, economic data.
- Cite credible sources: peer-reviewed literature, government reports, institutional data.
- Localize the need: National data sets the context, but local data makes it real to reviewers.
- Include community voice: surveys, focus groups, community advisory board input.

### Logic Model Components
```
INPUTS → ACTIVITIES → OUTPUTS → SHORT-TERM OUTCOMES → LONG-TERM OUTCOMES
(Resources)  (What you do)  (What you produce)  (Changes in 1-3 yrs)  (Impact in 5+ yrs)
```

Example:
| Component | Example |
|-----------|--------|
| Inputs | Staff, funding, curriculum, facilities |
| Activities | Conduct 20 training workshops, provide mentoring |
| Outputs | 200 participants trained, 200 certifications issued |
| Short-term Outcomes | 80% pass certification exam, 70% report increased skills |
| Long-term Outcomes | 60% employed in the field within 1 year, community health metrics improve |

### Evaluation Plan
- **Process evaluation**: Are you implementing as planned? (attendance, fidelity, dosage)
- **Outcome evaluation**: Are you achieving the intended results? (pre/post tests, follow-up surveys)
- Use validated measurement instruments when available.
- Mixed methods (quantitative + qualitative) strengthen the evaluation.
