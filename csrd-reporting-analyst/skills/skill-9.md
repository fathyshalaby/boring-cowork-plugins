# Gap Analysis and Roadmap Development
Structure a CSRD implementation project:

### Phase 1: Context & Gap Assessment
1. **Scope Definition**: Identify exactly which entities in the group architecture fall under CSRD (consolidated vs. individual reporting) and the transition timeline phase.
2. **Double Materiality Assessment**: The immediate first step. Determines reporting scope.
3. **Gap Analysis**: Compare the company's current reporting (e.g., existing GRI sustainability report, current HR data, current GHG inventory) line-by-line against the ESRS data points deemed material.
4. **Output**: A comprehensive list of missing policies, actions, targets, and data metrics.

### Phase 2: Design & Implementation
1. Establish a cross-functional ESG steering committee (Finance, Legal, HR, Operations, Procurement).
2. Draft missing policies and set required targets.
3. Implement data collection workflows and select IT systems.
4. **Dry Run / Mock Audit**: Produce a draft report for the fiscal year *before* mandatory reporting starts. Have internal audit or the external statutory auditor perform a mock assurance engagement to identify weaknesses without regulatory risk.

### Phase 3: Reporting & Assurance
1. Produce the final data.
2. Draft the Management Report sustainability section.
3. Apply XBRL digital tagging.
4. Complete the limited assurance audit process.
5. Board of Directors sign-off (Directors hold collective responsibility for the reporting).
